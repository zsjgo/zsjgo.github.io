一条SQL 语句的执行过程

Mysql 逻辑架构

首先给出MySQL 的基础架构图，

MySQL 可以总的分为 Server 层和存储引擎层两个部分。

Server 层内主要包括连接器、查询缓存、分析器、优化器、执行器等。

存储引擎层负责数据的存储和提取。其架构模式是插件式的，可以随意选择合适地存储引擎，Mysql 5.5.5 版本之后默认存储引擎为 InnDB。

查询语句 sql 执行流程

第一步，连接器

首先需要连接器连接数据库，连接器负责跟客户端建立连接、获取权限、维持和管理连接。

第二步，查询缓存

连接建立完成就可以执行select 语句，执行逻辑就会来到第二步：查询缓存。

缓存中存储的是之前执行过的语句和结果，存储形式以 key-value 对。

但实际查询缓存弊大于利，查询缓存会非常容易失效，只要对一个表进行更新，那这个表上所有的查询缓存都会被清空，因此查询缓存只适合用于静态表。

Mysql 8.0 版本直接将查询缓存整块功能删除。

第三步，分析器

分析器会做 词法分析 和 语法分析。

首先词法分析，需要识别出sql语句里的字符串分别都是什么，代表什么。比如从“select” 识别出这是个查询语句。

接着做语法分析，判断你输入的sql 语句是否符合Mysql 的语法。

第四步，优化器

Mysql 内部会对 sql 语句确定一个执行方案，比如当表里有多个索引的时候，优化器会决定使用哪个索引；或者在一个语句有多表关联的时候，决定各个表连接的顺序。

第五步，执行器

MySQL 通过分析器知道了你要做什么，通过优化器知道了该怎么做，于是就进入了执行器阶段，开始执行语句。

开始执行的时候，要先判断一下你对这个表 T 有没有执行查询的权限，如果有权限，就打开表继续执行。

至此，这个语句就执行完成了。
